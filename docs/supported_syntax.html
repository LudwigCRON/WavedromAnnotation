2.2 Supported Syntax¶

In order to let you choose the most appropriate format for you, examples are posted below:

Json¶

{ "signal": [
        { "name": "clk",  "wave": "p......" },
        { "name": "bus",  "wave": "x.34.5x",   "data": "head body tail" },
        { "name": "wire", "wave": "0.1..0." },
]}

Jsonml¶

in json-ml for strict compatibility with Wavedrom

{ signal : [
    // clock signal
    { name: "clk",  wave: "p......" },
    // bus data
    { name: "bus",  wave: "x.34.5x",   data: "head body tail" },
    // request signal
    { name: "wire", wave: "0.1..0." }
]}

Yaml¶

clk:
    wave: p......
bus:
    wave: x.34.5x
    data: head body tail
wire:
    wave: 0.1..0.

Toml¶

clk.wave = "p......"
bus.wave = "x.34.5x"
wire.wave= "0.1..0."

bus.data = "head body tail"

Note

Notice the group “signal” vanishes in yaml and toml syntax. Wavedrom used a primary group telling if this is a waveforms of signals, or registers, or logic.

However, this increase the indentation level, and is prone to error in yaml file format.

In toml file format, it increase the notation length as signal.<name of my signal>.wave, or using [signal] section without adding any advantages.
