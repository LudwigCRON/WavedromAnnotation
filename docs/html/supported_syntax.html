4. Supported Syntax¶

Undulate selects the syntax based on the extension of the file (at the exception of Json/Jsonml). However, each file has some specificities.

In order to let you choose the most appropriate format for you, examples are posted below:

Json¶

{ "signal": [
    { "name": "clk",  "wave": "p......" },
    { "name": "bus",  "wave": "x.34.5x",   "data": "head body tail" },
    { "name": "wire", "wave": "0.1..0." },
]}

Jsonml¶

in json-ml for strict compatibility with Wavedrom

{ signal : [
  // clock signal
  { name: "clk",  wave: "p......" },
  // bus data
  { name: "bus",  wave: "x.34.5x",   data: "head body tail" },
  // request signal
  { name: "wire", wave: "0.1..0." }
]}

Yaml¶

clk:
  wave: p......
bus:
  wave: x.34.5x
  data: head body tail
wire:
  wave: 0.1..0.

> notice the group “signal” vanishes in yaml

Toml¶

clk.wave = "p......"
bus.wave = "x.34.5x"
wire.wave= "0.1..0."

bus.data = "head body tail"

> notice the group “signal” vanishes in toml
